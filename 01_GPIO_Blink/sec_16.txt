SECTION 16
SPEED register : OSPEEDRy
00 : low speed, 01 : Medium speed, 10 : Fast speed, 11 : High speed
Data sheet의 OSPEED에서 각각의 속도의 주파수 확인 가능

Capacitance Load(CL) : GPIO가 충전해야 할 전기 저장량 - 클수록 느려짐, 단위는 pF (피코패럿)
VDD : 전원 전압 - 높을수록 빠르게 충전 가능

Bit-Banging : 하드웨어 I2C를 안쓰고 GPIO로 직접 I2C 신호를 만드는 것
GPIO로 일일이 HIGH/LOW 토글하면서 I2C 신호를 흉내
Trise : 상승 시간 - LOW -> HIGH
Tfall : 하강 시간 - HIGH -> LOW

!! OSPEEDR는 output 모드 일 때만 적용됨

pull-up/pull-down register : PUPDRy
역할 : GPIO 핀을 풀업(Pull-Up) 또는 풀다운(Pull-Down)으로 설정하는 레지스터
문제 상황 : Floating(떠있는 상태)에서 버튼을 안 눌렀을 때 값은 랜덤
- 전압 불안정
- 노이즈에 민감
- 0인지 1인지 알 수 없음

해결책 : 풀업/풀다운
Pull-Up : 핀을 VDD로 살짝 연결 - 버튼 안 눌럼 : HIGH, 버튼 눌림 : LOW
Pull-Down : 핀을 GND로 살짝 연결 - 버튼 안 눌림 : LOW, 버튼 눌림 : HIGH

사용:
버튼 입력 (풀업 권장)
LED 출력 (No pull)
I2C (풀업 필수!)
아날로그 입력 (No pull)
UART TX (No pull)
UART RX (풀업 권장)

Input Data Register(IDR) : 읽기 전용(눈)
외부 핀의 현재 물리적인 상태(전압)를 읽어서 저장해두는 공간
역할 : 핀에 전기가 들어오고 있나 확인
특징 :
- Read-Only(읽기 전용) : 개발자가 이 레지스터에 값을 쓸 수 없음. 핀의 전압 상태에 따라 하드웨어가 자동으로 값을 업데이트
- 실시간 반영 : 핀의 전압이 3.3V(HIGH)면 해당 비트가 1이 되고, OV(LOW)면 0이 됨
- 슈미트 트리거 통과 : 외부의 아날로그 전압이 '슈미트 트리거'라는 회로를 거쳐 깨끗한 디지털 0/1 신호로 변환된 뒤 이곳에 저장

Output Data Register(ODR) : 읽기 전용(손))
내가 핀을 통해 밖으로 내보내고 싶은 값을 저장하는 공간
역할 : "핀에 3.3V룰 내보내라"고 명령할 때
특징 :
- Read/Write(읽기/쓰기 기능) : 개발자가 코드로 1을 쓰면 핀으로 High 신호가 나가고, 0을 쓰면 Low 신호가 나감
- 메모리 기능(Latch) : 한 번 값을 쓰면, 다시 바꿀 때까지 그 상태를 계속 유지
- 읽기 가능 : ODR을 읽으면 내가 마지막으로 핀을 무슨 명령을 내냈는지 확인 가능(IDR과 달리 실제 핀의 상태가 아닌, 내가 명령한 값을 읽음)

IDR 경로: 핀(Pin) → 입력 드라이버 → IDR → CPU (읽기)
ODR 경로: CPU (쓰기) → ODR → 출력 제어 회로 → 핀(Pin)
